# 视频转 Atlas（GameDraft）

将角色动画视频在选定时间区间内抽帧，按固定单元格 **底边 + 水平居中** 对齐后拼成规则网格 PNG，并可选导出与运行时一致的 `AnimationSetDef` 草稿（`.anim.json`）及详细元数据（`.meta.json`）。

## 环境

- Python 3.10+

## 安装

在仓库根目录或本工具目录下执行：

```bat
cd /d d:\GameDev\GameDraft\tools\video_to_atlas
python -m pip install -r requirements.txt
```

## 启动

```bat
cd /d d:\GameDev\GameDraft\tools\video_to_atlas
python main.py
```

## 使用要点

1. **打开视频**，在**时间轴双柄条**上设置区段 **t0、t1**（粗选要抽帧的闭区间）；下方为读数。
2. 在 **「导出参数」** 中设 **目标 FPS、最大帧数**（及 **色键**，与导出一致），点 **「提取帧条」**：把 `[t0,t1]` 解码成横向 **缩略图序列**（帧条索引从 0 递增）。**生成 Atlas 前必须先提取**。
3. 在 **「导出首帧序号 / 尾帧序号」** 中选定要打进图集的 **连续片段**（含两端）。在缩略图条上辨认姿势后再改序号，不必再在视频画面上对首尾。
4. 点 **「生成 Atlas …」**：仅导出该序号范围内的帧；`.meta.json` 中会带 `stripSliceFrom` / `stripSliceTo` 记录帧条上的起止索引。
5. **源画面预览**：Qt Multimedia 播放；「仅循环播放所选区间」等仍可用。可选 **「首帧半透明叠层」**在播放器下方做 OpenCV 混合预览（辅助对尾帧，非必须）。
6. **单元格宽高**：每帧画布尺寸；角色 **底对齐、水平居中**，与游戏中 `anchor (0.5, 1)` 思路一致。
7. **列/行**：`0` 为自动；`col = index % cols`，`row = index // cols`。
8. **帧编号基准**：anim JSON 可选从 **0** 或 **1** 起。
9. 生成后可 **循环预览生成动画**；状态栏有导出片段首尾 MSE 提示。
10. **最大帧数**：`0` 不限制；若限制且小于按 FPS 在区间内可采帧数，会在 **整段 `[t0,t1]`** 上均匀下采样后再进帧条。

### 自动对齐循环首尾（减少接缝跳变）

- **姿势优先（默认）**：指纹在 **非透明 alpha 包围盒** 内裁出角色、等比缩放到 64×64 再算 MSE，更利于找到 **首尾姿势接近**、适合做成可循环帧动画的区间。建议开启 **「对齐计算使用色键」** 以得到干净 alpha。若关闭界面上的 **「姿势优先」**，则退回 **整帧** 缩小比较（接近早期整图行为）。
- **按当前长度对齐首尾**：先大致拖出想要的**区间长度**（t1−t0），点该按钮；工具会顺序解码一遍视频、在时间上平移该长度，使「起点帧」与「接近 t1 的最后一帧」的指纹 **MSE 最小**，并写回时间轴。
- **高级：搜索区间与长度**：在「最小时长～最大时长」内按「长度步进」枚举多种长度，对每种长度做上述平移搜索，取全局 MSE 最小的一对 (t0,t1)。可先拖好 t0、t1，再点 **「用当前区间填最小时长/最大时长」**，按间隔 L=t1−t0 一键填入（并夹紧在视频时长内）。
- **采样步长**：指纹时间采样间隔；填 **0** 表示自动（约不超过 0.05s）。步长越小越准、越慢。
- **匹配在干什么**：对齐在 **离散** 时间点上做。对每个候选起点 `t0` 与区间长度 `L`，令 `t1=t0+L`，在采样时刻里找 **不超过 `t1` 减半帧** 的 **最后一个** 时刻作为 **尾帧**，与 **首帧** 的 64×64 指纹算 MSE，MSE 最小的 `(t0,t1)` 写回时间轴。素材本身若没有「姿势接得上」的循环，MSE 会一直偏大，只能尽量找相对最接得上的点，而不是「匹配失败就随便跳到最后一帧」；若尾时刻总像贴在全片末尾，可把 **采样步长** 略缩小（时间线更密），并确认 **顺序读** 已包含真正最后一帧（工具已补最后一帧指纹）。
- **对齐计算使用色键**：勾选时与下方关键色/容差一致；不勾选则用原画比较。
- 完成后若 MSE 仍高于内置建议阈值，状态栏会提示可能仍有可见跳变，可缩小采样步长或收窄高级搜索范围。

## 导出文件

- `*.png`：Atlas 图（RGBA，若启用色键则含透明）。
- `*.meta.json`：列行、帧时间、索引等调试信息。
- `*.anim.json`（可选）：GameDraft `AnimationSetDef` 草稿；请将 `spritesheet` 改为游戏中真实 URL 路径，并把 PNG 复制到 `public` 下对应位置。

## 区间循环仍觉跳变时（资料与建议）

Qt/QMediaPlayer **做不到真正无缝**区间循环（类似 [QTBUG-34706](https://bugreports.qt.io/browse/QTBUG-34706)、[StackOverflow 讨论 gapless / flushMode](https://stackoverflow.com/questions/70007728/qt-multimedia-is-there-a-way-to-implement-gapless-loop-video-playback-in-pyside)），反复 `setPosition` 会触发 H.264 **关键帧对齐**，易产生短暂冻结或闪一下。可尝试：

- 用 **ffmpeg** 重新导出一段「快 seek」素材，例如全帧内或极短 GOP：`x264-params keyint=1:min-keyint=1:scenecut=0`（体积会变大，利于循环点跳转）。
- 保证 **首尾画面**本身接得上（与自动对齐工具一致）；否则再顺滑的解码也会像「跳」。
- Windows 若 `main.py` 里 `QT_MEDIA_BACKEND=windows` 仍不理想，可临时去掉该行试 **ffmpeg** 后端（或反之），行为因机而异。
