非图 YAML 说明（15 个）

这些文件没有 schema: chronicle_sim_v3/graph@1 的「计算图」语义，不会在 csim cook run 里当主图解析；与模拟的关系是「被图内节点读」或「初始化 Run 目录」。

根目录（5）

  • context_schema.yaml：全工具链的 key 命名空间与路径规则，改它等于改「世界如何落盘」的契约，需与 engine/keymap、读节点对齐。
  • event_types.yaml：事件类型定义（条件、cooldown、权重、周期等），week 里 director/GM 侧提示文本来自 eventtype.format_for_prompt。
  • pacing.yaml：全局面板；图里 read.config.pacing 可带 preset 参数，或读默认文件。
  • rumor_sim.yaml：rumor.bfs_engine 的默认 params 来源，也可用 read.config.rumor_sim 选预设。
  • style_fingerprints.yaml：叙事/文风指纹，供需「按条改写」的节点或后续工具引用。

预设 presets/（6）

pacing/ 与 rumor_sim/ 各三套；图 YAML 里 params: preset: default 等即指向此目录下的文件（具体解析以引擎与节点实现为准）。

模板 templates/*.example.yaml（4）

供 csim run init 生成用户 Run 下的 config/providers.yaml、llm.yaml、agents.yaml、cook.yaml 的初始内容，不属于模拟 DAG。
