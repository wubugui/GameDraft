# 交付与管线（全体 Agent 必读）

本任务由编年史模拟器子进程采集结果：**管线以 Cline 进程的标准输出（stdout）为主交付物**；若你在工作区用工具写了文件，**仍须保证最终可被采集的内容完整、可解析**。

## 必须遵守

1. **禁止**仅用「已完成」「已按要求撰写」「全文共 N 段、约 N 字」等**元叙述**代替**正文**；若任务要求产出叙事、JSON、结构化列表，则 **stdout 的最终可见输出**必须包含**可直接使用的完整正文或可解析 JSON**。
2. **若使用 `write_file` / 编辑工具** 将长文写在 `output.md` 等文件中，**仍须在最终回复中**给出完整正文（或确保 stdout 即为完整 JSON），**不要**假设「写在文件里」即等于已交付——管线会尽力从工作区文件保底回读，但**以 stdout 为首要依据**。
3. **JSON 类任务**（导演、GM、NPC 意图等）：优先写入 **`agent_output.json`**（或 initializer 约定的种子文件名），内容须为**以 `{` 或 `[` 开头的合法 JSON**；勿用纯摘要替代。
4. **叙事/周纪类任务**：**最终回复**须为**完整叙事文本**（多段中文），勿只写完成说明。

## 工作区说明

每次调用使用**独立**的临时工作目录（UUID），**不会**覆盖你之前任务的历史；历史会归档到 Run 目录供审计与人工回溯。
