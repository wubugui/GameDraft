# 角色照明实验室(伪世界 RT / irradiance cache)

单张场景背景图 → 全自动深度 → 免锚点免平面标定 → 补全伪世界几何 → 角色照明。
实现对应推导文档《伪世界角色照明:从精确辐射传输到全部近似》(A1–A9)。
位置:`tools/character_lighting_lab/`(2026-07-21 自实验目录转正;`out/` 数据不入库)。

## 用法

```bash
./dev.sh char-lighting                      # → http://localhost:5311/(自动开浏览器)
./dev.sh char-lighting -- --no-open --port 5311
```

**烘焙全部在软件内完成**(①重建区,异步任务队列 + 实时日志,可排队可离开页面):

- **⟳ 应用参数重烘当前场景**:改任何重建参数后点它;
- **＋ 新场景入库…**:选一张背景图 → 按当前重建参数全自动跑完整管线
  (深度→标定→mesh→体素→SAM门控→probe→碰撞),完成自动进下拉;
- **⟳ 重烘全部**:按各场景已保存参数逐个排队重烘。

CLI 仅作脚本化备用:`./dev.sh char-lighting -- --build <图.png> --name 名字 [参数…]`。

查看器:方向键/WASD 移动角色;Tab 循环 / 1‑4 直选照明模式;右侧全部参数实时;
「重建参数」区改俯角/ppu/EV/凸出阈值后点重建(调用 /api/rebuild,秒级)。

## 四个照明模式

| 模式 | 实现 | 对应近似 |
|---|---|---|
| RT  | 逐片元 cosine 半球采样,体素网格内步进,miss→镜像查 J̄ | ref 路径,(8)+(14) |
| L1  | probe 三线性 + 4 系数球谐辐照度 | A9 一阶 |
| L2  | probe 三线性 + 9 系数球谐辐照度 | A9 二阶(≈无损,Ramamoorthi) |
| BIN | probe 三线性 + octahedral 8×8 法线 bin 辐照度(无球谐截断) | A9 纯数据 |

实测四模式收敛:temple 烛边 RT 5.5 / L1 4.6 / L2 5.5 / BIN 5.4(8bit luma)。

## 管线阶段(pipeline.py)

1. **深度**:Depth Anything base(项目 .tools venv),按图内容哈希缓存。
2. **标定(免平面/免锚点)**:视差仿射模型 d=1/(a·raw+b),粗细网格搜索
   最小化地面掩码上的鲁棒 |∇Y|²("地面平均尽量水平",真实起伏保留);
   地面掩码 = 朝上表面 ∩ 洪泛连通画面底边,迭代 2 轮;o 使地面中位高=0。
3. **地面场**:地面高度 Y_g Laplace 延拓全图 → 非平面行走面 d_walk(角色脚吸附)。
4. **补全**:灰度闭运算找凸出物 + 组件厚度充气;背景层深度/颜色扩散 inpaint;
   行走面之后沿射线填实(地下);→ 占据+辐射体素(192×h×64, RGBA16F, a=占据)。
5. **辐射**:HDR 统一模型(零手调:双尺度 log 亮度残差 + 白天闸门 + 发光体置信度)。
6. **J̄ 闭合**:3 个自由点全球面 gather,命中归一化,radiance SH L2;miss 镜像查询。
7. **probe**:18×10×8 网格,实心内 probe 无条件吸附最近自由体素(EDT),
   196 Fibonacci 方向 gather → 同时产 L1 / L2 / bins 三种 cache。
8. **可走/碰撞**:地面掩码 − 膨胀的凸出物脚印,PNG 输出,查看器洪泛取最大连通域出生。

产物在 `out/<场景>/`:manifest.json、volume.bin、probes_*.bin、front_depth.bin、
walk_depth.bin、walkable.png、audit.png(标定/分层/体素审计图)。

## 已知边界

- 角色显示亮度是严格物理(A1:立绘=albedo),暗场景角色就是暗;曝光 β 是设计旋钮。
- 白天室外无方向太阳(hero light 层未做,对应 A8 的解析灯,后续项)。
- 遮挡判定用可见壳深度(front_depth),角色只与第一层皮比较。
- 体素分辨率下薄结构(细栏杆)会变粗;occluder 厚度是启发式。
- 本工具全部文件在实验目录内,未触碰项目本体。

## v2 变更(对第一版验收意见的修复)

1. **probe 改世界空间体积**:网格轴对齐世界坐标(x × 高度带 × 深度),覆盖
   "地面 .. 地面+2.6wu" 角色可达带;着色时 quad 像素 q→世界(mat3)→三线性。
2. **折叠追踪(A7 局部化)**:朝相机的射线镜像 qz 后真追局部几何,J̄ 只兜真 miss——
   probe E 空间方差从 ≈0 升到 0.11~0.30,移动角色光照真实变化。
3. **世界空间碰撞**:世界 XZ 地面网格(非平面 y 场),blocked = 占据柱测试 +
   "可见表面高出地面>0.45"(空壳建筑覆盖判定);与屏幕遮挡完全解耦——
   角色可走进建筑背后被正确遮挡而不被禁止移动。
4. **起伏增益 relief**(默认 1.8):把结构相对钉死地面的深度残差放大,补偿单目
   模型对竖直对比的压缩(城镇建筑从 0.5wu 恢复到 ~1wu+)。
5. **3D 检视页签**:轨道相机(拖拽旋转/滚轮缩放/Shift平移),点云(可见壳/补全层/

6. **射线可视化**:角色胸口 40 根 cosine 采样射线,黄=直接命中/橙=折叠命中/蓝=miss,
   2D 与 3D 视图都画。
7. **法线**:法线显示模式(注意朝相机 n.z=−1 显示为橄榄色,不是淡紫)、
   压平滑杆(隆起→纯平插值)。
8. **HDR 面板**:P50/P99 nit、白天分、发光像素%、实际最大增益 + 前后 log2 亮度
   直方图;增益图可叠加背景;max_gain_ev 进重建参数。
9. **quad 整体缩放滑杆**;重建按钮置顶吸附、参数改动呼吸提醒。

## miss 闭包控制(运行时旋钮,不用 rebake)

probe 烘焙把每份 cache 拆成三件套存储:E_hit(命中贡献)/ E_amb(miss 方向的 J̄ 贡献)
/ cov(余弦加权命中覆盖率)。运行时:

- **miss环境强度** s:E = E_hit + s·E_amb(s=0 即 miss=黑,严格模式);
- **miss不计入**:E = E_hit / cov(hitnorm 归一化);
- RT 模式同两策略逐射线执行。

注意:**折叠开着时 miss 集合≈空,这些旋钮天然无效**(洞已被折叠填上,是特性不是 bug)。
要看闭包差异:重建区取消「烘焙折叠」重烘(~10s),此时朝相机半球全是 miss,
四个模式的旋钮全部生效(实测 temple:s=0→0.4,s=1→2.5,s=2→4.4,归一→1.5,
且 RT≈L2≈BIN)。当前 temple 为无折叠烘焙(旋钮可玩),雾津街头为折叠烘焙(推荐画质)。

## 3D 检视:mesh 与手性

- 世界矩阵 M 第三行取反(det=−1 正交阵):q 空间(x右/y上/z朝里)是左手系,
  纯旋转会把左手性带进世界,GL 右手相机里必然镜像。取反后世界为右手系,
  转置=逆与等距性均保留;方向键映射同步翻转。已数值验证:画面左点→3D 屏幕左。
- 重建导出真三角 mesh(mesh_verts/mesh_idx,~7-9 万三角形):可见壳/补全层/
  地面延拓三层共一套顶点布局(pos3f+rgb+tag),深度跳变 >0.55q 的三角形被丢弃,
  形成的洞=未观测区域的诚实表达。3D 页「网格」勾选切换 mesh/点云,层开关共用。

## NEE 直接光(发光体显式采样)

- **分账**:HDR 恢复天然可分 `全量 = base + emit`(emit=发光增量)。base 只走余弦
  半球(画=收敛解,间接光已含);emit 只走 NEE——增益图×深度提取发光 surfel
  (连通域聚类,大组细分,功率截断≤48),逐光源可见性追踪 + 解析投影。
  余弦射线命中发光体素只记 base——零重复计数,完美分割免 MIS。
- **烘焙四分账**:probe atlas 列布局 [base+cov | amb | emitray | nee],
  「NEE直接光」为纯运行时开关(RT 与三种 cache 同步语义,无需 rebake)。
  atlas 合并是因为 18 个独立采样器超 GL 上限 16。
- **发光体为各向同性 surfel**(无发光端余弦):火苗高度场法线朝相机,单面朗伯
  会把侧面贡献掐零;且射线路径的发光体素本就方向无关,两边必须同物理。
- **期望守恒**:开关 NEE 均值几乎不变是分账正确性的体现(无偏分割),NEE 买的是
  低方差与确定性;效果随 1/r² 集中在贴近光源的部位(脚边烛火照亮腿部)。
  要放大观感:重建区 HDR最大EV 拉到 4–5 重烘。
- 「光源」勾选:2D 橙十字(大小=功率)/3D 橙点,直接检视提取结果。

## 语义门控与双 quad(v2.4)

- **SAM3 语义门控回归**(`semantic_gate.py`,重建参数「语义门控」默认开):
  发光体置信度 = 统计模型 × SAM3 物体证据(semantic mask×score + core/halo/direction
  组件门,均为实验原公式移植)。首次每场景推理约 4–5 分钟,按图哈希缓存
  (`out/<scene>/sam_gate_<hash>.png`)之后零成本。实测 temple 光源 48→3
  (只留真烛火群)。模型快照已迁入 ~/.cache/huggingface(2GB,防 /tmp 蒸发);
  必须 HF_HUB_OFFLINE(SOCKS 代理会弄死 huggingface_hub 的在线检查)。
- **遮挡/着色双 quad**:遮挡用平行相机的 billboard(整张 sprite 与脚点深度比较,
  = 游戏 sprite 排序惯例);着色继续用世界直立 quad(受光位置正确)。
  直立 quad 参与深度测试会让上半身穿透前景物,两个代理各司其职后消失。

## 排错备忘(真踩过的)

- probes_*.bin 布局 (probe, coeff, rgb),纹理必须 width=coeff、height=probe。
- probes_valid.bin 存 0/255(R8 归一化);shader 阈值 step(0.002, v) 双兼容。
- GLSL ES 点精灵必须显式写 gl_PointSize,否则 ANGLE/Metal 复用陈旧值(全屏巨点)。
- 法线可视化颜色:朝相机 n.z=−1 → 蓝通道≈0(橄榄色),别当成 bug。
- 内嵌浏览器面板后台会节流 rAF——真浏览器前台无此问题(实测前台 34fps)。
