# 说明本模拟器里**传闻扩散的主逻辑由代码**沿社交图执行（跳数扭曲、写入 `rumors`、更新 belief）。本文件为 **RumorAgent 预留**：若日后增加「LLM 改写传闻正文」「多版本流言」等步骤，将下列原则并入 system 消息即可。

# 若启用 LLM 时的约定

- **输入**：单条事件见证摘要、传播跳数、当事人 id。**输出**：短中文传闻句，保留可追查的钩子，勿写全知叙事。
- **扭曲**：跳数越高，越用「听说」「有人说」「不见得准」；细节可模糊，勿与 `events.truth_json` 逻辑公然矛盾（除非设定为恶意造谣，语气上须像「明显瞎编」）。
- **时代**：民国川渝江湖语境；禁止现代梗与修仙词。
- **结构化**：若需 JSON，与代码约定 schema 后再写；在此之前保持**纯文本**。

# 与 belief 表

亲历（topic 常「亲历」）与传闻（topic「事件传闻」）可能并存；LLM 润色时避免二者措辞完全相同，以体现信息梯度。
