# Godot 运行壳完全对齐复审

> 日期：2026-07-12  
> 权威运行时：TypeScript / Pixi  
> 被审计运行时：`godot_port/`  
> 判定口径：同一资源与 JSON、同一初态、同一输入/命令序列，应得到相同逻辑状态、事件副作用、存档载荷与可操作玩家路径。

## 1. 当前结论

当前版本已经从“确定存在 P0 逻辑分叉”推进到：

- 已知的 action、场景、叙事、存档、Water、Sugar、菜单/地图/档案等确定性分叉已修复；
- TypeScript→Godot→TypeScript→Godot 双向存档 E2E 通过；
- 共同命令序列逐步检查点与最终快照 `field differences: 0`；
- macOS 导出包可真实启动并响应 runtime bridge，Windows 导出包通过 PE/PCK 完整性校验；
- 22 场景主线/支路、20 个过场、真实 UI 按钮/键盘路径均有 Godot 测试覆盖。

但仍**不能标记“完全对齐”**。当前主要阻断已从“已知逻辑错误”转成“证明面与像素/逐帧表现尚未闭合”。若要求同数据在两壳得到完全可互换结果，下面第 3 节仍必须完成。

## 2. 本轮已对齐的确定差异

### 2.1 状态与时序

- `fadingZoom` 参数、fade 默认值、camera baseline、`changeScene` 缺省坐标、`moveEntityTo` 兼容归一。
- action 失败会停止后续批次；Scenario/Narrative 的 action 边界与嵌套 drain 顺序对齐。
- Scene switch 改为串行队列；onEnter 内重入仍 fire-and-forget，避免环形等待。
- 同场景 cutscene staging 只重建绑定实体，不再整场 reload；不额外触发 scene enter/ready、音频、zone 和相机副作用。
- 初始 onEnter、initialCutscene、dt clamp、相机 follow 状态、巡逻资格对齐。
- GraphDialogue deferred chain 使用单 runner、代际取消和最终 end 兜底；禁用提示优先级对齐。
- Scripted dialogue 与 encounter 的重复启动按 TypeScript 覆盖当前会话。

### 2.2 存档与互通

- 恢复事务只发一次 `save:restoring`；Quest/Archive 全事务 restoring guard。
- 读档前静默清 active zone，旧时间线 `onExit` 不污染目标档。
- `dialogueLog`、真实 `playTimeMs`、全局随机流状态参与保存/恢复。
- 两端菜单均有玩家可达 JSON 导出/导入入口。
- 双向真实恢复流程通过：TS→Godot→TS→Godot。

### 2.3 随机与小游戏

- Water degraded 过滤方向修正。
- Water 实体、拉取面板和 Sugar 氛围统一为 UTF-8 FNV-1a seed + xorshift32，并统一消费顺序。
- `randomBranch` 使用两端共享、可存档的全局 xorshift32 流，不再是 `Math.random()` 对 `randf()`。
- Water 失焦 held、压力长按起始边沿/坏参数语义已对齐。
- PaperCraft 调色板、拖拽预览和槽位使用真实部件 PNG，不再是只加载后丢弃的文字壳。

### 2.4 UI、过场、音频和照明

- 菜单、地图、背包、规矩、档案、书籍均有真实 Button 路径；测试不再只靠 debug 方法。
- 菜单 newGame/returnToMain/resume 事件状态栈已接线。
- 档案 `[img:...]` 实际加载图片；InspectBox 动态高度；Numpad 数字选择可用。
- 过场 120ms 防误触、双帧 arming、fade settle、跨场景音频基线、parallax easeInOut、Ken Burns clamp 已对齐。
- 音频 fade 改为线性振幅插值，不再线性插值 dB。
- 照明 probe 缩至 96px 并低通采样；ambient、0.4×实体高度采样、遮挡提前结束、hotspot 着色策略已按 TypeScript 调整。

## 3. 仍未完全对齐/未充分证明

### 3.1 强差分覆盖仍太窄（验收阻断）

当前 parity 共同序列只有：setFlag、移动玩家、激活位面、切换场景。它能抓住切场 settled tick 与 onEnter 分叉，但没有覆盖：

- 对话推进/选项、encounter、cutscene skip、三个小游戏输入；
- 101 个 action 的成功、缺参、坏参、失败、中止和嵌套路径；
- save→继续运行→load→再次 save 的逐检查点全状态；
- 鼠标、触屏、失焦、按住/松开边沿；
- 并发切场、读档中断过场、销毁中的在途 action。

快照也仍未完整纳入全部 save bucket、active zones/action queue、UI overlay 返回栈、音频集合与 fade 中间值、cutscene/minigame 在途摘要和 RNG 子流位置。

### 3.2 缺少统一事件 trace（验收阻断）

当前比较最终/检查点状态，但没有比较单调序号事件流。两端可能最终 flag 相同，却以不同次数或顺序发出 `dialogue:end`、`quest:completed`、`scene:enter`、`zone:exit`。需要记录并比较：事件名、规范化 payload、action id、transition id、tick/frame。

### 3.3 渲染仍不是逐像素同算法

已对齐主要公式，但以下仍是不同实现：

- Pixi `BlurFilter quality=3` 与 Godot shader/mipmap 低通不是同一采样核；
- planar/deferred shadow 的 softening 仍由不同渲染管线实现；
- probe 的 Pixi blur pass 与 Godot mip LOD 只是目标一致，不是逐采样一致；
- 字幕 speaker/body 分色、字体度量、换行、movie-band 精确布局仍非同一排版器；
- Godot Control 与 Pixi Text 的字体栅格、hinting、亚像素和 GPU 混合不可假定逐像素一致。

因此尚未建立场景、过场、小游戏固定尺寸 screenshot/frame-sequence golden，也没有量化允许误差。

### 3.4 固定时间与音频输出未闭环

- 两端都 clamp `dt<=0.1`，但还没有共享 fixed-tick 驱动所有移动、相机、动画和小游戏的差分试验。
- 音量包络公式已对齐，但解码器、重采样、设备 mixer 不同；尚无音频集合/包络检查点或离线 waveform golden。

### 3.5 异常/损坏资源路径未穷尽

Godot GDScript 没有与 TypeScript `try/catch/finally` 完全同构的异常模型。合法内容路径已有 teardown 测试，但损坏贴图、损坏动画、action handler 脚本错误、读档打断 minigame/cutscene 等路径尚未逐项证明输入锁、staging、policy depth 和 active session 必然归零。

另外，真实图对话和脚本对话集成测试虽然功能断言通过，但单独运行时 Godot 分别报告 29 和 18 个 `ObjectDB` instance 在退出时泄漏。完整 runner 当前只把 `SCRIPT ERROR` / `ERROR` 视为失败，尚未把该 warning 升格成 teardown 门禁。

### 3.6 Windows 仅验证包，未在 Windows 真机运行

macOS 导出包已真实启动；Windows 当前只在 macOS 上验证 PE 头、大小和嵌入资源。要声称 Windows 可交付，仍需 Windows 真机启动、导入/导出存档、主路径和 GPU shader smoke。

### 3.7 权威内容本身仍有 60 条 warning

这些不全是 Godot 迁移错误，但会扩大“两壳对坏数据如何回退”的不确定性。当前高风险项包括：hotspot 引用未进入 index 的 `cutscene_10`、叙事画布声明信号与真实对话图 emit 漂移、空 inspect hotspot、未登记 flag，以及多位面场景 transition 未声明 planes 归属。若验收口径要求任意同一份 JSON 都同结果，则必须清零这些 warning，或为每类无效输入补两端相同的拒绝/降级契约与差分用例。
