## 本轮新发现并修复的确定性分叉

1. 自动化命令曾在启动路由完成前消费，测试场景会被随后到达的 `dev_room/teahouse` 覆盖。现在 `runtimeReady` 只在 ticker 和启动路由落地后开放，runner 必须等待该字段。
2. 玩家/NPC 动画帧游标未进入共享快照，原“0 差异”掩盖了 204 个帧号、余量和像素密度开关差异。加入 `entityVisualState` 后逐项修正，现为 0。
3. Pixi 与 Godot 的纹理像素中心相差半像素。Godot 相机只在最终栅格层补偿 X `-0.5px`，逻辑投影和点击换算保持原值。
4. JavaScript `Math.round(-19.5)=-19`，Godot `round(-19.5)=-20`。像素吸附改为显式 `floor(v+0.5)`，消除负半整数地图整屏 1px 偏移。
5. Pixi 过滤器使用预乘 alpha，Godot CanvasItem 使用直 alpha。Godot 遮挡分支原先同时缩 RGB/A，合成时等效重复压暗；现只缩 alpha，并移除错误的预乘 RGB clamp。
6. BGM 淡化曾由 Howler 墙钟和 Godot 帧 Tween 分别驱动，掉帧时包络阶段不同。Godot 改为单调墙钟结算；测量器也同时记录两端真实 elapsed（定时器只保证“至少等待”），再将斜率归一到 250ms，避免把调度延迟误判成包络差异。
7. 资源图从 745 增到 747 后测试写死旧数量并在断言后挂起；现按资源图 `summary.localTargets` 动态核对并保留最低规模保护。
8. `雾津街头` 的找活 zone 已移动，主线 E2E 仍写死旧坐标；现按 zone ID 从当前 JSON 多边形计算中心，22 场景无 debug 主线再次通过。
9. TS 开启实体像素密度匹配时仍使用 Pixi 默认线性采样，Godot 原先错误切成 nearest；高密度背景场景的人物和道具因此稳定过锐。现统一为 linear，并用测试锁住启停语义。
10. Pixi `BlurFilter.strength` 是最终渲染像素单位，Godot shader 原先把同一数值当源纹理 texel。高分辨率道具缩小时半径可小十余倍；现按 source-texel/screen-pixel 换算。
11. 高密度背景缩小时两端单级线性采样核仍有稳定差异。Godot 现只对 source/screen 大于 1 的背景应用按 X/Y 缩小率推导的低通；放大背景严格透传，27 场景扫描防回退。
12. 糖轮与纸艺虽然几何一致，Pixi/Godot 的离屏合成仍有半像素采样相位差。Godot 现使用通用 CanvasGroup 复合 shader 在最终采样层补偿 X 半像素，不改逻辑坐标、命中区或 JSON。
13. 纸艺原先套用了通用容器布局，工作台、素材栏、卡片和按钮没有执行 TypeScript 的两套自适应公式。现逐项复刻 560×410 工作台、250 宽素材栏、缩放/定位/槽位/按钮公式，并以共享 JSON 坐标驱动命中。
14. 四类小游戏此前只有各自行为测试，没有统一证明“代表运行帧 + 内部状态”同时一致。现水域、糖轮、纸艺、憋气都进入同一个 visual/state sweep；离散字段严格相等，仅渲染遥测浮点允许 `1e-9` 数值容差。
15. 最终强 parity 曾发现两处合同漂移：Godot 已输出 `minigameDebug` 而 TypeScript 正式快照未输出；TypeScript 新增 `signal.unlistened` 与叙事存档 migrations 后 Godot 未同步。现两端实现、32 字段快照 Schema 和 Godot 契约测试全部补齐，重新回到 0 字段差异。
