## 运行时行为模型总览（审查建立的理解）

- **启动**：`main.ts` 解析 URL 参数（`mode=dev` / `play_cutscene` / `devScene` / `narrativeWarp` / 三种小游戏 preview）→ `Game` 构造即装配全部系统并统一 `init` → `start()` 依次 renderer/strings/config → 创建 UI → 分发条件上下文工厂 → 注册 ~90 个动作 → 接线 InteractionCoordinator/EventBridge → 并行 loadDefs → 按模式进 dev_room 或初始场景 → 挂 ticker。
- **游戏循环**：`tick(dt)` 按 `GameStateController.currentState` 分支（Exploring 跑 nav/交互/zone/NPC；Dialogue/Encounter/Minigame/UIOverlay/Cutscene 各只跑对应 UI），随后统一跑气泡/相机/逐实体深度遮挡与光照/阴影/Y 排序。暂停语义靠 `overlayReturnStack` 压/弹。
- **数据流**：状态唯一存 FlagStore；条件统一走 `evaluateGraphCondition`（5 类叶子）；行为统一走 `ActionExecutor`；叙事为 5 层信号驱动（世界→对话 emitNarrativeSignal→scenario→主线 flow→quest 镜像），`NarrativeStateManager` 单队列 drain、每图每信号至多一条迁移。
- **场景切换**：`SceneManager.switchScene` 经 `sceneSwitchTail` promise 队列串行：saveCurrentSceneMemory→fadeOut→unload→load（实体实例化受 sceneMemory/过场 staging 过滤）→onEnter 动作批→`scene:ready`（Game 在此附加深度滤镜、启动巡逻）→fadeIn。
- **UI 层**：主体是 Pixi Container（非 DOM；仅 DebugPanel dock 与触控层是 DOM），open 全量 build、close 全量 destroy；写操作走 EventBus，读操作走注入的 Provider 接口。
- **存档**：`SaveManager` → localStorage 三槽，v1 格式；load 为「存活实例灌 deserialize + 重载场景」，不重建系统。

## 一、P1 — 玩家可见错误 / 软锁 / 存档完整性

### 1.1 拾取与实体显隐（同一根因，系统性）

**R1. 已拾取热点下一帧复活，可无限刷道具** `src/systems/InteractionSystem.ts:105-116` + `src/entities/Hotspot.ts:250-251,330-331` 【CONFIRMED，主线程复核】
`markHotspotPickedUp` 仅 `setInactive()`（active=false），而 InteractionSystem 每帧对所有热点无条件 `hotspot.setEnabled(base)`（`setEnabled` 直接写 `this.active = enabled`），base 不查 pickedUp → 拾取后下一帧热点复活，可无限重复拾取刷钱/刷道具。且切场景时 `saveCurrentSceneMemory` 按 `!active` 反推拾取状态，复活后 active=true → **拾取状态永远进不了 sceneMemory 和存档**。encounter 型热点（`hotspot:pickup:done` 自消费）同样复活。

**R2. `setEntityEnabled` 隐藏的 NPC 对话结束即弹回** `src/core/ActionRegistry.ts:726` + `src/systems/InteractionSystem.ts:119-130` 【CONFIRMED】
会话级 `setEntityEnabled` 只改实体不写 memory；回到 Exploring 第一帧被每帧 `setVisible(base)` 打回可见。演出内正常（InteractionSystem 不跑）、结束即穿帮。只有 `persistNpcEntityEnabled/persistHotspotEnabled` 能存活。
**根因（R1/R2 共享）**：InteractionSystem 每帧把「派生基底」强写回实体显隐，覆盖一切瞬时运行态。修复应让显隐回写尊重运行态位（pickedUp/会话 override），而非各处打补丁。

**R3. 存档不 flush 当前场景运行态 → 读档物品复制** `src/systems/SceneManager.ts:1394-1411`（serialize 直读 memory）、`saveCurrentSceneMemory` 仅 switchScene 调用 【CONFIRMED】
拾取后未切场景就存档 → memory 里没有本场景拾取记录 → 读档后热点重生而物品仍在背包。独立于 R1（修好 R1 后本条仍成立）。

**R4. 条件隐藏的 pickup 被误记为已拾取，永久丢失** `src/systems/SceneManager.ts:1231-1235` 【机制 CONFIRMED / 触发需「pickup+conditions」数据，当前无】
`!active && type==='pickup'` 即记入 pickedUp——因条件不满足被隐藏的 pickup 在切场景时被永久标记拾取并入档，条件满足后也不再实例化；若是推进必需道具 → 软锁。

### 1.2 对话 / 演出流

**R5. 嵌套 `playScriptedDialogue` 的 `dialogue:end` 被误判为整段图对话结束** `src/core/InteractionCoordinator.ts:135-138,225-244` + `src/systems/DialogueManager.ts:83-89` 【CONFIRMED，现网内容可触发】
图对话 runActions 内嵌 playScriptedDialogue 是受支持写法，但其收尾无条件 emit `dialogue:end`，InteractionCoordinator 两处监听无来源判别：NPC 路径→NPC 恢复巡逻中途走开+镜头拉远；inspect 路径→提前跑 `data.actions`、强切 Exploring，剩余图对话期间玩家可走动、打字机冻结。**现网触发链**：`寻狗_看进山路.json` 节点 `n_night`（经 `阎王岭山口.json` 热点 graphId 进入）。

**R6. deferred 链式图对话在 Exploring 态播放** `src/systems/GraphDialogueManager.ts:610-640` + `src/core/EventBridge.ts:54-56` 【CONFIRMED】
`endDialogue` 先 emit `dialogue:end`（EventBridge 置 Exploring）再内部启动 deferred 队列图——不经 action 处理器，无人再 `setState(Dialogue)`。接续图整段在 Exploring 播放：玩家自由走动、可触发热点、打字机不更新。demo 编排脊椎（对话信号→narrative onEnter→startDialogueGraph）正是此模式。

**R7. WithTrace 求值器缺 `reached:true` 分支，switch 分支语义分裂** `src/systems/graphDialogue/evaluateGraphCondition.ts:289-296`（对照 182-194） 【CONFIRMED，主线程复核；两分区独立命中】
非 trace 版支持 `reached:true → hasReachedState`；WithTrace 版只比 `getActiveState === stateId`。而对话图 **switch 节点与图 preconditions 只走 WithTrace 版** → 线上 **27 处** switch case 的 `reached:true`（如 `寻狗_守夜续油`、`寻狗_茶馆吹牛`）在状态被越过后全部错判 false，玩家看到"事前"版台词；F2 条件追踪同样显示错误结果误导排查。

**R8. 图对话纯路由环无护栏，主线程死循环冻死页面** `src/systems/GraphDialogueManager.ts:646-734` 【护栏缺失 CONFIRMED / 需坏数据触发】
`drainUntilBlocking` 对 switch/ownerState/contextState 的 `continue` 无步数上限；坏连边（A→B→A）→ 同步 while 死循环。CLAUDE.md 明示校验抓不到图内 next 连边，此处是最后防线，应加迭代上限。

**R9. 过场 skip 后被放弃的演出轨残留** `src/rendering/CutsceneRenderer.ts:347-376` + `src/systems/CutsceneManager.ts:600-619` 【CONFIRMED】
`abortCutsceneOps` 只结算当下挂起的 op，被打断的 async 演出函数继续跑并新建不受管控的 op：①非 parallel 步跳过后剩余 tween/wait 按真实时长跑完才收尾（跳过表现为卡住数秒）；②parallel 步 race 放行后，被放弃 track 的 `finalizeStill/showImg` 在 cleanup **之后**把 Sprite 加回 cutsceneOverlay（残留图带进正常游玩画面）；`showTitle` 对已置 null 的容器解引用抛未处理 rejection。关联：skip 时 parallel 在途 track 的 ActionExecutor 动作无取消机制（`CutsceneManager.ts:600-619`），过场后对已 destroy 的 `_cut_*` 演员操作。

**R10. 过场中读档：步进机不停、挂起 promise 悬死演出通道** `src/systems/CutsceneManager.ts:914-930` 【逻辑 CONFIRMED / 可达性依赖存档入口门控】
`deserialize` 只重置标志；`executeSteps` 中断条件不查 `playing`，剩余步继续在读入的新世界上执行；若正停在 waitClick/showDialogue，监听已 unsub、promise 永不 resolve → `cutscene:end` 不发，演出通道悬死。

### 1.3 状态机 / 遭遇 / 规矩

**R11. `ruleUse:apply` 无 `resultText` 时 UIOverlay 无解软锁** `src/core/EventBridge.ts:112-124` + `src/ui/RuleUseUI.ts:183-190` 【CONFIRMED，两分区独立命中；latent——当前数据恰好都带 resultText】
KeyF 开面板压栈进 UIOverlay；`selectSlot` 直接 `close()` 不弹栈不复位；EventBridge 只在 `if (p.resultText)` 分支内恢复状态，而 `resultText` 是 optional 字段 → 一旦策划省略，结算后永驻 UIOverlay：移动禁用、面板打不开（canOpen 要求 Exploring）、Esc fallback 也要求 Exploring。

**R12. 遭遇选项可生成空集且 UI 无退路 → 软锁** `src/systems/EncounterManager.ts:106-196` + `src/ui/EncounterUI.ts:294-311` 【机制 CONFIRMED / 触发依赖内容全条件化】
所有选项被规矩/条件过滤后 emit 空数组，Options 相无 Esc/离开出口，`endEncounter` 只由 `encounter:resultDone` 触发。引擎无兜底（空集自动结束或注入默认「离开」项）。

**R13. `startEncounter` 传未知 id 状态卡死在 Encounter** `src/core/ActionRegistry.ts:409-412` + `src/systems/EncounterManager.ts:92-96` 【CONFIRMED】
先 `setState(Encounter)` 再查 def；查不到 warn 后 return，不发 `encounter:start`，唯一恢复路径 `encounter:end` 永不到来。与 `startCutscene/switchScene` 的失败即恢复模式不一致。

**R14. 场景 onEnter 含 `changeScene` → 切场队列环形死锁，永久黑屏** `src/systems/SceneManager.ts:1101-1110,1216-1220` 【机制 CONFIRMED / 当前数据未用此写法】
loadScene 内 await onEnter 动作批，onEnter 内 changeScene 排进 `sceneSwitchTail` 队尾等前一个 job，前一个 job 等 onEnter → 环形等待；fadeIn 永不执行，之后所有切场请求全部冻结。编辑器允许 onEnter 挂任意 action，属一触即死地雷。

**R15. `setZoneEnabled` 触发全量 zone 重注册，活跃 zone 被强制 exit/enter 重放动作** `src/systems/ZoneSystem.ts:58-68` + `src/systems/SceneManager.ts:370-374` 【CONFIRMED】
任何 `setZoneEnabled(Persistent)`（哪怕针对无关 zone）→ 玩家所在 zone 的 onExit 立刻执行（气味掉帧、RuleOfferRegistry 被 clear），下一帧重进重放 onEnter（伤害/通知/叙事信号等一次性语义重复）。「进 zone 后禁用自身」的常见写法也会触发自身 onExit。

### 1.4 叙事状态 / 存读档

**R18. 读档失败仍提示成功 + load 无原子性** `src/core/SaveManager.ts:68-92` + `src/ui/MenuUI.ts:215` 【CONFIRMED】

**R19. 读档后 HUD 全部陈旧** `src/core/Game.ts:2452` + `src/systems/InventoryManager.ts:172` + `src/systems/QuestManager.ts:273` 【CONFIRMED】

**R20. 主菜单「新游戏」不重置任何状态，直接以旧局继续** `src/core/EventBridge.ts:100` 【CONFIRMED，主线程复核】

## 四、系统性根因（修复应按根因组织，而非逐条打补丁）

1. **每帧派生回写覆盖运行态**（R1/R2/R4）：InteractionSystem 把「条件+基底」推导的显隐每帧强写回实体，任何瞬时运行态（拾取、会话隐藏）都会被冲掉。需要一个「运行态优先级」模型（pickedUp/会话 override > 派生基底）。
2. **同一语义多份实现漂移**（R7、R17、D1、D2、B21）：trace/非 trace 求值器、三份动作参数表、三份小游戏骨架、两种滚轮坐标换算——凡复制必漂移，已全部漂出真 bug。需收敛为单源或加一致性测试。
3. **「结束事件」语义过载**（R5、R6、B13）：`dialogue:end` 同时承担脚本台词结束/图对话结束/状态恢复三种含义，嵌套场景必然误判。事件应带来源/会话 id，或状态恢复只认会话发起者。
4. **serialize/deserialize 不对称**（R3、R16、R18、R19、D4 + P3 多条）：不清空就恢复、恢复了不发事件、存了进行态读回丢弃——存读档是当前最脆弱的横切面，建议做一次「存→读→比对全系统状态」的自动化往返测试。
5. **失败路径不恢复状态**（R11、R13、B10、B12）：setState 在前、可失败操作在后、无 finally 恢复，是软锁的统一形状。约定「先验证再切状态」或统一 try/finally 恢复模板。
6. **dev 设施门控不一致**（T1–T5）：有的用 `import.meta.env.DEV`、有的用 `?mode=dev`、有的裸奔。需要一次门控盘点 + 约定唯一判据。
