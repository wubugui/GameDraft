## 系统性根因(修复时按族治,别单点补)

1. **「绕过 staging 直写源数据」族第二次复发** — 2026-07-02 修过 P1-25(出生点拾取弹窗 re-snapshot),本轮 `_delete_selected`/`_add_spawn_at`/`_undo_entity_refactor` 三条同族路径漏网(P0/P1 三条)。根因是"直写源+stale staging 快照盲提交"这个模式没有护栏测试;修复时必须给 `_commit_pending_scene_edits` 补"直写前先 commit 或 re-snapshot"的流程探针,否则第三次长回来。
2. **登记面镜像失配,parity 只锁存在性不锁语义** — 五处手工镜像的历史教训重演:CONTENT_ID_PARAMS↔控件生成链(3 个漏网裸文本框)、_PARAM_SCHEMAS↔叙事 required(25 处倒挂拦保存)、KNOWN_DIRTY_BUCKETS↔LSP overlay_payloads(characterRegistry 漏镜像)、signal_refactor 两张登记表(注释宣称的 parity 测试不存在)、重构引擎 speaker 通道↔运行时真实数据形状(sceneNpc 整通道漏)。每一处修复都应连 parity 测试一起落。
3. **引用生命周期(rename/delete)防护极度不均** — 同一工作区里三种档次并存:entity_refactor 引擎(全跟随+报告+撤销)/character·smell(计数警告或拒删)/**flag·plane·quest id·rule 碎片·出生点 key 完全裸奔**。且面板 id 输入框裸改可完全绕过重构引擎。恰恰 flag 是历史 P0 的域。
4. **fail-open 惯性:取不到状态就当没事** — 叙事 Web F5 无脏检查、React 无 ErrorBoundary(白屏后 Save All 假成功)、脏检查超时当"不脏"、`_nav_hit_generic` 不看结果就报"已定位"、音频试听失败全静默。方向应统一为 fail-safe:拿不到状态当脏、失败要说话。
5. **Save All 的连坐面过大** — 图对话面板 flush 失败 raise 中断全局(其它编辑器连 flush 都没执行);presave 引用/scenarios 校验不按脏桶收口,盘上既有的无关坏数据锁死一切保存;plane extends 成环、water 坏 id 都拖到 Save All 才爆且全域被拦。策划的体感是"一个角落的错,谁都别想存"。
6. **协作模型(agent 写盘+编辑器常开)零外部修改防护** — 全库唯一例外是图对话(_loaded_disk_bytes 基线比对);其余全部 last-writer-wins 静默覆盖。这是本项目标准工作流,不是边缘场景。
7. **预览可信度漂移** — 水域画布精灵尺寸与运行时不同口径且无视 displaySize、扎纸画布坐标系/底图语义与运行时脱节、糖画 chargeButton 首屏默认值错位。画布"看起来对"是策划敢用的地基。
