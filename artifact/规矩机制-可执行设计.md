# (a) 规矩机制 可执行设计 —— 不踩"解谜不讲理 / 违规即死"两大雷区

> 对照真实实现：`rules.json`（象/理/术 + verified）、`encounters.json`（type:rule 门控）、`pressure_holds.json`（⑨喊名长按）。
> 对标：山海旅人「逆梦改命」招牌机制、《寻找伪人》「学情报→识别→选择→后果」同构循环。

## 0. 现状诊断（基于你仓库里真实数据）

**已经做对的（保住别动）**
- 象/理/术三层 + `verified`（unverified→effective/questionable）= 规矩会"从传闻升级成亲历确认"，这正是规则怪谈受众要的"可推理 + 细思极恐"。
- `fragments[]` 按 `ruleId`+`layer` 拼合，多源（说书人/手札/茶馆小二）凑出一条规矩——天然的探索奖励。
- 遭遇里 `type:"rule"` 选项用 `conditions:[{flag:"rule_X_acquired"}]` + `requiredRuleId` 门控：**学过规矩才解锁安全选项**。
- `ghost_encounter` 四个选项（硬上/[规矩]躲观察/扔桃木/跑）**全部存活、无 Game Over**，只是结局风味与 flag 不同——这已经是正确范式。
- 阿秀暗线已经缝进规矩文本（"认不准他是个什么""另有什么替他把场面圆上了"）——埋得好。

**目前偏薄（要补的）**
1. 规矩只有 6 条，且多数只有 `xiang` 一层 → "三层"结构还没真用起来。
2. 遭遇选项的差异**只停在 resultText + flag**，没有真正的下游后果（拿没拿到安全结局，对后面几乎无影响）→ 玩家会觉得"选啥都一样"。
3. "违规/选错"目前要么吓一跳照样过（ghost），要么 abort 后逃跑回城（喊名）→ 还没有"代价有重量、但不劝退"的中间档。
4. 没有"提示阶梯"：规矩文本一次性给全，或者没给，缺"象=免费线索→理=推理→术=应对"的渐进投喂。

## 1. 四条核心原则（一票否决级）

1. **可推理 > 玄学**：任何遭遇的"正解"，玩家在到达前必须**能从场景/对话/档案拿到线索**。绝不出现"喂豆包给指嘴的鬼"那种全凭脑洞（纸嫁衣被骂的原话）。
2. **违规有后果 ≠ Game Over**：违规走"叙事化代价阶梯"（见 §3），让违规**有戏可看**，而不是黑屏读档。死亡只保留给极少数明确预警过的硬节点。
3. **象/理/术 = 内置提示系统**：三层本身就是渐进提示，别另做 UI 提示按钮（§2）。
4. **规矩长满世界**：一条规矩要在"传闻→遭遇→长按/演出→档案回响"四处复用（§4），而不是用一次就扔。

## 2. 把象/理/术做成"讲理的提示阶梯"

把三层重新定义为玩家视角的**信息梯度**，而不只是文本分层：

| 层 | 玩家拿到时的状态 | 来源 | 在遭遇里的作用 |
|---|---|---|---|
| **象** | "听说有这么回事"（unverified） | 免费/低成本：说书、闲聊、告示、环境观察 | 解锁"知道这里不对劲"，但**不保证安全**——给的是警觉，不是答案 |
| **理** | "我懂为什么了"（亲历后 effective） | 中成本：一次遭遇/一段推理后回写 | 解锁**安全选项**（`type:rule`），并把 `verified` 升级 |
| **术** | "我知道怎么应对"（可执行，可能 questionable） | 高成本：道具/手法/特定前置 | 解锁**更优结局**或**反制**（消耗道具、特殊分支） |

**讲理三铁律：**
- **象层必须可免费获得**：进任何危险遭遇前，玩家至少能在场景里捡到/听到该规矩的"象"。没有象就进死局 = 不讲理。
- **理/术可缺，但缺了只是"更险"，不是"必死"**：没学理→只能硬上（惊吓但存活）；学了理→可安全；有术→可反制。三档对应 §3 的代价梯度。
- **`verified` 状态当"做对了"反馈**：玩家正确应用规矩后，把对应 layer 从 unverified→effective，并在档案里留一句"亲历确认"。这就是即时正反馈，解决"开放解读 vs 明确反馈"的冲突——主线给明确反馈，留白放进档案/支线。

## 3. "违规有后果"的代价阶梯（替代 Game Over）

每个遭遇的非最优选项，按这张梯度给后果，**全部用现有 flag/状态系统就能落地**：

| 档 | 触发 | 代价形态（叙事化） | 落地方式（现有能力） |
|---|---|---|---|
| **T0 惊吓无损** | 硬上但本就无致命 | 吓一跳、风味文本，照常推进 | `setFlag` 记风味分支（已是 ghost_encounter 做法） |
| **T1 资源/状态损** | 莽撞、用错道具 | 丢道具/铜钱、留"惊魂"状态影响后续选项 | `consumeItems` 反向 + 一个状态 flag 作后续 `conditions` |
| **T2 关系/线索损** | 违反禁忌、得罪人 | NPC 关系变冷、某条线索/碎片**这周目拿不到** | flag 关掉某 `giveFragment` 入口；NPC 对话分支变 |
| **T3 分支改道** | 喊名应声、动了不该动的 | 不死，但**走更难/更黑的支线**（如逃亡线、被跟） | 现成：喊名 `onAborted→hanming_answered→forest_escape_run` 就是 T3 |
| **T4 死亡/重来** | 极少数硬节点，**且事前明确预警** | 真死，回最近检查点 | 仅留给"明知山有虎"型、且象层已强提示过的点 |

**设计配额建议**：一周目里 T4 ≤ 2 次，其余全部 T0–T3。**让违规更多是"故事走歪了"而不是"你输了"**——这正是 Disco/寻找伪人 让人愿意试错的原因。

> 直接可改的例子：`ghost_encounter` 的"硬着头皮走上去"现在是 T0（吓一跳照过）。可升一档到 T2——硬上后留一个 `ghost_spooked` flag，让后面城隍庙叫魂(⑪)那段**阿秀的反应更激烈/多一句暗示**，让"莽撞"在后文有回响。零新系统，只加 flag + 一个对话分支。

## 4. "长满世界"的复用结构（让规矩撑得起 8–12 章 / 450h）

一条规矩 = 一个**跨场景的内容包**，按这个模板量产（与你 demo 基线的"单元制作模板"对齐）：

```
规矩包: rule_no_answer_name（山里喊名别应）
├─ 象  : 传闻来源 ×2–3（说书人/背尸老手/手札碎片）→ fragments[]
├─ 理  : 一次亲历遭遇（⑨喊名 pressure_hold）→ 触发后 verified: li=effective
├─ 术  : 应对手法（闷头别应 / 反走 / 借阿秀）→ 解锁更优 onComplete
├─ 遭遇: 至少 1 处 type:rule 门控选项（看进山路）
├─ 回响: 档案"见闻录"留一条亲历记录 + 阿秀暗线一句
└─ 复用: 同一规矩在后续章节换皮复现（神仙顶路上再遇喊名，但更狠）
```

**复用纪律**：每条主线规矩至少出现 **2 次**（首遇学会 + 后段加难复现），让"懂规矩"产生**跨章节的能力增长感**（替代 RPG 升级）。这是把"无战斗无养成"做出成长曲线的关键。

**量产经济**：象层(fragments) 用 GPT 批量写初稿最省力、最能"铺满世界感"；理/术层因为要接遭遇/长按，必须人工验收。所以**铺量靠象、扣质靠理术**。

## 5. 对标借鉴（3 条可直接抄）

1. **山海旅人「逆梦」→ 你的「规矩」当招牌动词**：山海旅人每关核心是"逆转一个梦改命"。你应让**"用一条规矩破一个局"成为每章的标志性时刻**，并在 UI/演出上仪式化（学会术的瞬间给一段有镜头/光影的小演出回报）。山海旅人续作从96%崩到60% = **招牌机制不深化就会疲劳**，所以每章的规矩破局必须有新花样，不能复制粘贴。
2. **《寻找伪人》「情报会变微妙/有误导」→ 你的规矩要有"存疑/会骗人"档**：你已有 `verified: questionable`（镇尸四样"不知哪样起作用"）。把它用足——**故意放几条会误导的规矩/传闻**（象层说的和理层亲历的不一致），让玩家学会"质疑传闻"。这制造"可推理却不被喂答案"的瘾，也呼应阿秀"暗中圆场"的真相。
3. **《寻找伪人》多结局后果 → 你的规矩账要在结局结算**：哪些规矩学全了/用对了/被骗了，汇到 2–3 个结局的判定里（呼应你"结局是命门"）。不是"对错评分"，是"你这一路怎么走的"。
