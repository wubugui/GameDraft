# 策划 Graph 内容操作指南

目标：编辑内容仍在现有编辑器里做；生产工作台只负责检查、追踪、验收、Debug 和素材生产。

当前交付验收状态见 `docs/plan/production-workbench-acceptance-status.md`。

## 1. 打开

1. 打开工程 `D:\GameDraft`。
2. 双击 `start-production-workbench.cmd`。
3. 或在命令行运行：

```powershell
npm run planner:gui
```

4. 也可以直接运行 Python 入口：

```powershell
.\.tools\Python311\python.exe -m tools.production_workbench D:\GameDraft
```

## 2. 每天开始

1. 进入 `每日检查`。
2. 点击 `运行每日检查`。
3. 正常通过时，报告里会看到 `Python 编辑器/Narrative smoke`、`生产工作台 smoke`、`Python import smoke`、`TS Narrative/runtime save smoke tests`。
4. 如果有 `error` 或 `blocker`，先点 `复制报告` 给 Codex 修。
5. 只有 warning 时可以继续做内容，但要看 warning 是否和今天要做的剧情有关。
6. 每日检查、剧情单元自检和验收相关报告会自动保存到 `resources/editor_projects/editor_data/production_workbench/reports`。
7. 如果按钮显示“运行中 / 加载中”，先等它结束；后台任务没结束时工作台会阻止切换工程或关闭窗口。

## 3. 做一个剧情单元

剧情单元不是新的剧情编辑器。它只是把一个剧情块变成“可追踪、可验收、可复制报告”的工作包。

执行逻辑只有三层：

1. `这个剧情单元是什么`：给人看的定义，不驱动游戏。
2. `自动验收怎么跑`：会被转换成 runtime debug 命令，发给正在运行的游戏。
3. `结果、阻塞和素材需求`：记录这块现在能不能过。

日常填写步骤：

1. 进入 `剧情单元`。
2. 左侧选择要做的剧情单元。
3. 不确定当前该做什么时，先点顶部 `操作向导`。它会按当前状态告诉你下一步点什么。
4. 需要看更详细的问题明细时，再点 `当前单元自检`。
5. 在 `1. 这个剧情单元是什么` 里填：
   - `类型`：点 `选择`，不要从下拉框里翻。
   - `制作状态`：点 `选择`，按当前真实进度选。
   - `剧情入口`：玩家怎样开始这块内容，例如 `进入码头后和铁环男孩对话`。如果想让自动验收复用，也可以写 `scene:test_scene spawn:door`。
   - `剧情出口`：做完后应该到什么状态，例如 `ringboy_flow.done`。
   - `通过标准`：一句话说明怎样算过，例如 `发出 ringboy.met，任务 bridge_find_source 进入 Active`。
6. 在 `2. 自动验收怎么跑` 里填：
   - 不要手写 ID，不要手填 `999` 这类草稿。
   - 看上方 `验收路线` 表，它会显示当前起点、前置、执行步骤、选项、期望结果和复查项。
   - 如果不知道怎么开始，先点 `生成验收草稿`。它会从当前剧情单元涉及的 dialogue、signal、state、quest 推一条起步路线。
   - 草稿不会覆盖已有路线；生成后必须人工看一遍，再点 `1. 检查脚本`。
   - 用下方按钮添加内容：`选场景`、`选对话`、`加 flag`、`NPC`、`热点`、`点坐标`、`移动坐标`、`路径`、`选 option`、`加 signal` 等。
   - 所有会变多的 ID 都走搜索选择器，不走下拉，不手写。
   - 坐标类步骤会先选场景，再在场景背景图上点击取点；路径就是按顺序点多个点。
   - 添加错了就在 `验收路线` 表里选中那一行，点 `删除选中`。
   - 顺序错了就选中那一行，点 `上移` 或 `下移`。
   - 要发给 Codex 看，只复制路线时点 `复制验收路线`；要复制全部上下文时点顶部 `复制当前单元报告`。
7. 点击 `1. 检查脚本`。
8. 没有 error 后，再去原编辑器改具体对白、状态、quest、素材。

可以自由写人话的字段只有这些：

- 显示名
- 剧情入口
- 剧情出口
- 通过标准
- 阻塞
- 素材需求
- 备注

自动验收区是机器生成脚本显示区，正常不手写。

制作状态规则：

- `未做`：还没开始。
- `制作中`：正在做，可以先不补齐验收路线。
- `可玩`：已经能走一遍，但还没正式验收。
- `待验收`：必须补齐入口、出口、通过标准和验收路线，否则保存会拦住。
- `通过`：必须最近一次验收结果为 `通过`。
- `冻结`：通过后不再轻易改。

`素材需求` 和 `备注` 旁边也有选择器：

- `选已有素材需求`：从当前 runtime 素材里搜索，作为重抽/修改参考。
- `加 zone 到备注`：从场景 zone 里搜索，不手写 zone id。

## 4. 运行验收

1. 先运行游戏：

```powershell
npm run dev
```

2. 打开浏览器里的游戏页面。
3. 回到生产工作台 `剧情单元`。
4. 点击 `2. 发送到游戏运行`。
5. 等运行中浏览器执行命令。
6. 点击 `3. 完成并记录结果`。
7. 结果不对就点 `复制报告` 给 Codex。
8. 通过后把制作状态改成 `通过` 或 `冻结`。

## 5. 看 Graph 问题

1. 进入 `Graph诊断`。
2. 默认看 `全部`；只看某个剧情单元时点 `选择范围` 搜索选择。
3. 重点看：
   - `Signal flow`
   - `Flag / Action read-write`
   - `Quest dependency`
   - `Dialogue route explain`
   - `State direct write / 风险`
   - `Runtime trace timeline`
4. 看不懂或结果异常，点 `复制报告` 给 Codex。

## 6. 运行时 Debug

1. 先运行 `npm run dev` 并打开游戏页面。
2. 进入 `运行时Debug`。
3. 点击 `刷新运行时快照`。
4. 需要复现前，点击 `清 runtime trace`。
5. 需要立刻抓状态，点击 `请求 runtime 快照`。
6. 出问题时点 `复制事故报告`。

## 7. 素材生产

1. 进入 `素材审计`，点击 `刷新素材审计`。
2. 需要风格参考时点击 `生成风格/命名参考`。
3. 进入 `素材任务`。
4. 类别、操作、透明要求都点 `选择`，不要手写；输出目录、宽高、帧数、参考素材、具体要求再按需求填写。
5. 点击 `生成任务文本` 看是否准确。
6. 点击 `执行 Codex 并记录`。
7. 需要自动缩放/裁透明边时勾选 `执行后自动生成 _ready 后处理副本`。
8. 生成结果会自动保存 prompt、日志、savedPath、token 摘要和验收结果；原始 JSONL 只在 run 目录里留给开发排查。
9. 生成后进入 `素材候选`，点击 `刷新候选`，查看 savedPath、验收结果和候选状态。
10. 单张调整去 `图片工具`；动画 sheet 去 `动画Sheet`。

## 8. 常用验收路线例子

1. 点 `选场景`，搜索并选择起点场景。
2. 点 `NPC`，搜索并选择要交互的 NPC。
3. 点 `对话`，搜索并选择 dialogue graph。
4. 点 `走完对话`。
5. 如果需要坐标，点 `点坐标` 或 `移动坐标`，在场景图上点位置。
6. 如果需要走路线，点 `路径`，按顺序点多个点。
7. 如果有选项，点 `选 option`，搜索并选择已有选项文本。
8. 点 `加 signal`，选择必须出现的 signal。
9. 点 `加 state`，选择必须到达的 narrative state。
10. 点 `加 quest`，选择 quest 和目标状态。
11. 点 `存读档` 或 `重进场景`。

## 9. 不做的事

- 不直接手写复杂 JSON。
- 不维护 CSV/YAML 第二套生产管线。
- 不在生产工作台里编辑具体剧情字段。
- 不把 warning 当成一定不能做内容，但 error/blocker 要先处理。
