# 关二狗故事 10 月交付范围与排期基线

日期：2026-05-31

## 结论

目标口径必须锁定为：

**2026 年 10 月 31 日交付一版关二狗主线完整可通关、关键支线可玩、素材够用的完整版本。**

不把目标定义为商业级精修版，不把 10 月当主制作月。

判断：

- 10 月 1 日完成：不现实。
- 10 月 31 日完成可通关完整版本：可以冲，但必须严格控范围。
- 10 月 31 日完成丰满精修版本：不现实。

## 范围上限

为了在 10 月底完成，制作范围必须压在以下上限内：

| 类别 | 上限 |
| --- | ---: |
| 主线章节 | 8-12 |
| 主要场景 | 8-12 |
| Dialogue graphs | 25-40 |
| 重要支线 / 小情景 | 6-10 |
| Narrative graphs | 12-20 |
| Scenario lines | 8-14 |
| Quest lines | 8-15 |
| 结局 | 2-3 |
| 新玩法系统 | 0，最多 1 |

超过这些上限时，优先砍支线、砍演出、砍素材精修，不砍主线可通关。

## 制作口径

技术：

- 现有编辑器继续作为唯一制作入口。
- Runtime JSON 仍是权威数据格式。
- 代码全部交给 AI 写，人只做验收和决策。
- 只修阻塞制作的问题，不继续扩主工具链。

内容：

- 人工在编辑器里完成编排、对话、状态、任务、场景配置。
- GPT 负责拆稿、生成初稿、批量改写、检查引用。
- 每个剧情单元必须有验收方式：能进、能出、能重进时状态正确。

素材：

- GPT image2 抽卡生成。
- 不细扣单张质量。
- 素材只按“能表达、风格大致统一、无明显穿帮”验收。
- 音频优先氛围和关键音效，不逐段精修。

## 月度门禁

### 6 月：锁范围和第一章可玩

必须完成：

- 关二狗完整章节拆分。
- 主线状态总图草案。
- 第一章从进入到完成全流程可玩。
- 核心角色、核心场景、核心素材风格样张确定。

禁止：

- 大改管线。
- 新增非必要玩法系统。
- 在未锁主线前扩支线。

### 7 月：主线 60%

必须完成：

- 主线前 60% 的场景、对话、状态链进入编辑器。
- 关键角色的 owner wrapper 建立。
- 至少 3 条重要小情景完整可玩。
- 每周至少一次从新档开始通关当前主线。

门禁：

- 如果 7 月底主线不到 60%，砍掉一半支线。

### 8 月：主线全通

必须完成：

- 主线从开局到结局至少一条线全通。
- 所有必须场景和必须对话进入工程。
- 支线只保留最有价值的 6-10 条。
- 素材全部达到“临时可交付”。

门禁：

- 如果 8 月底主线不能全通，9 月不再新增任何支线和玩法。

### 9 月：内容冻结

必须完成：

- 所有主线、支线、小情景内容冻结。
- 只允许修 bug、补断点、补明显缺失提示。
- 完成至少 3 次完整通关验收。

门禁：

- 9 月 15 日后不新增剧情分支。
- 9 月 30 日后不新增素材需求，除非缺失会导致流程不可理解。

### 10 月：收尾和交付

必须完成：

- 完整通关测试。
- 修阻塞 bug。
- 替换最丑或最穿帮素材。
- 补关键音效和氛围。
- 打包一个可演示版本。

禁止：

- 新系统。
- 新支线。
- 大规模改写主线。
- 追求素材精修。

## 单元制作模板

每个剧情单元都按这张表登记：

| 字段 | 填写要求 |
| --- | --- |
| 单元 ID | 稳定 id，不随显示名变化 |
| 显示名 | 策划可读名称 |
| 类型 | 主线 / 支线 / 小情景 / 结局 |
| 场景 | 涉及哪些 scene |
| Dialogue graph | 涉及哪些对话图 |
| Narrative State | 读写哪些 graph/state/signal |
| Scenario | 涉及哪些 scenario phase |
| Quest | 涉及哪些任务 |
| 素材 | 背景 / 角色 / 道具 / 音频 |
| 验收 | 从哪里进入，怎样算完成 |
| 预计小时 | 人工编辑器制作时间 |
| 状态 | 未做 / 制作中 / 可玩 / 冻结 / 通过 |

## 砍项规则

当排期落后时，按顺序砍：

1. 砍非关键支线。
2. 砍支线内分支。
3. 砍演出细节。
4. 砍素材变体。
5. 砍非必要场景。

不得优先砍：

- 主线闭环。
- 玩家理解因果所需的信息。
- 状态链验收。
- 保存 / 重进后的流程正确性。

## 每周验收

每周末固定做一次：

- 从新档开始跑当前主线。
- 记录第一个阻塞点。
- 记录所有状态错乱点。
- 记录所有找不到入口/出口的内容。
- 下周优先修这些问题，再做新内容。

通过标准：

- 当前已标记“可玩”的单元可以从入口进入。
- 完成后状态正确。
- 重进场景后状态仍正确。
- 对话分支能按 Narrative State / Scenario / Quest 正确变化。

## 最终交付标准

10 月底的“完成”只按以下标准判断：

- 主线至少一条完整路线可从开局通到结局。
- 所有保留支线都能进入、完成或明确失败退出。
- 没有阻塞通关的运行时 bug。
- 没有明显缺失导致玩家无法理解的关键素材。
- 存档读档后主线、支线、NPC 状态不乱。
- 有一份已冻结的内容范围清单。

不以以下标准判断：

- 素材是否精美。
- 音频是否逐段定制。
- 所有原始设想是否都做完。
- 所有支线是否丰满。
- 所有演出是否精修。
