# Godot 运行壳迁移原则

## 完成定义

Godot 工程不是独立内容分支，也不是玩法裁剪版。`public/assets/**` 与
`public/resources/runtime/**` 始终是唯一内容源；TypeScript 运行时是迁移期间的行为权威。

只有同时满足以下条件，才可称为搬完：

1. Godot 能消费现有全部内容 JSON，无需 Godot 专用字段或手工二次配置。
2. Action、Condition、图对话、叙事图、场景、实体、系统存档和生命周期语义与 TypeScript 一致。
3. 现有 UI、过场、音频、渲染效果、小游戏和调试命令均有 Godot 等价实现。
4. 全量内容契约测试通过，两个运行壳在相同命令序列下产生等价状态快照。
5. 关键场景完成真机视觉与交互回归；不存在只靠编译或局部冒烟推断全量完成的项目。

## 兼容审计

`tools/audit_runtime_coverage.py` 每次从 TypeScript 权威实现和当前内容重新提取能力面，
再与 `compatibility/capabilities.json` 中已经验证的 Godot 能力对账。

参数级与字段级盘点输出到 `compatibility/authoritative-contract.json`。当前合同包含
101 种 Action 的参数、31 种运行命令、29 个存档键、场景/实体实际字段、780 个对话节点、
20 段过场的步骤统计、44 个数据文件的 3,187 个字段路径及责任归属、runtime 媒体清单。

`compatibility/resource-graph.json` 记录内容/代码引用到本地媒体的完整边、每个导出候选文件的 hash，
并跟进动画 manifest、overlay 短名和动态对话立绘 bundle。它只生成清单，不复制或改写源资源。

开发中查看报告：

```bash
python3 godot_port/tools/audit_runtime_coverage.py --write-report --write-contract
```

最终验收门：

```bash
python3 godot_port/tools/audit_runtime_coverage.py --strict
```

在实现、语义测试和真机验证没有齐全之前，不得把能力写入 `capabilities.json`。
