# 偏差记录 inbox

实践锚定知识(method / decision)唯一的保鲜来源。任何 agent 在工作中发现
**现实与库内文档打架**(某阶段白做了、判断依据误报、发现新死路、库里缺了该有的卡),
收尾时在这里丢一条记录。**写入零门槛,不需要任何审批**——错过现场即不可还原。

治理 run 负责蒸馏这里的积压:改稿、加死路、标疑或升审批题。蒸馏完成的记录由治理 run
删除并计入 governance-log。

## 格式

文件名:`YYYY-MM-DD-<slug>.md`

```markdown
---
target: animation-matting     # 相关文档 id;库里还没有对应文档时写 missing
date: 2026-07-11
session: 可选,会话/任务标识
---

现象: 一行——工作法说 X,实际发生了 Y。
证据: 一行——文件/命令输出/artifact 指针,让蒸馏者能复核。
建议: 可选一行——你认为该怎么改(治理 run 只作参考,不照抄)。
```

三行写不下的事,说明它是工作产物不是偏差记录——放 `artifact/`,这里只留一行指针。
