# agent_docs 宪法(收录标准与治理规则)

> 本文件是 `agent_docs/` 的最高规则。治理 run、入库判断、审批分级都以此为判据。
> 修改本文件属于"不变量级变更",必须经人审批。

## 0. 定位:项目理解的缓存

`agent_docs/` 是**对项目理解的缓存**,面向所有 agent(Claude、Codex 及未来任何客户端)公有。
ground truth 永远在两处:**代码**(机制事实)与**实践史**(工作法有效性)。本库只存
不随实现漂移的东西:不变量、指路牌、工作形状、配方、决策。治理 run 的本质是用一次
新鲜理解重建缓存,而不是维护一份会腐烂的架构描述。

**治理体系的元方法**(2026-07-11 制作人拍板):用**结构性的 agent 方法**去治理项目方法——
正确性建立在**结构**上(盲重建防确认偏误、对抗验证、最小审批面、机械门禁),不建立在
把事情拆到步骤上。本库与全部治理流程都**只提供元方法**(疆域、判据、结构保证),
不提供具体方法;具体手法至多以"可选提示"存在,"怎么做"由执行 agent 的现场智能补齐。
治理流程自身的写作与重审同样受此约束。

**收录检验标准**:一条内容若依赖具体类名、方法名、步骤序或字段清单才能成立,它是
实现描述,不收;类名只作举例或锚点出现。会漂移的事实要么写成生成区块、要么带
authority 锚点,二者皆无不收。

## 1. 五类知识(类型轴)

| 类型 | 存什么 | 过期方式 | 检索钥匙 | 写作高度 |
|---|---|---|---|---|
| **norm 规范** | 不变量+过程义务+门禁+红线,每域一篇 | 只能经人审批修改 | 任务类型(域) | 性质表述,零实现细节 |
| **method 工作法** | 被实践验证的工作形状:阶段骨架+判断点+分工+死路 | 实践偏差蒸馏(事件溯源) | 任务形状(tasks) | 写判据不写步骤,写死路不写正路细节 |
| **mechanism 机制卡** | 每机制一张:一句话定位、权威源指针、硬契约、已知坑、验证方式 | 治理 run 代码对账 | 文件路径(paths)+主题 | 指路牌+契约,限一页,超了即架构描述 |
| **recipe 配方** | 实测能跑通的具体操作(命令/端点/参数) | 实测失败即标疑;代码对账 | 主题+任务 | 可以写细——确定性真实存在的地方才配写细 |
| **decision 决策记录** | 带日期的已拍板事项,**含被否方案** | 永不修改,只被新决策取代 | 主题 | 一页内:背景一段、决定一句、被否路线列表 |

**method 的解剖**(六栏):①适用时机;②阶段骨架(每阶段=目的+完成判据,不写怎么达成);
③判断点(哪里要现场判断、拿什么证据判);④分工契约(程序做什么/agent 裁什么/人拍什么板);
⑤已知死路(链 decision);⑥向下指针(确定性部分链 recipe/机制卡)。

**method 三原则**(2026-07-11 制作人拍板;违反任一即不收,存量卡治理 run 按此重审):
- **疆域不地图**:只圈边界与判据,不铺步骤;agent 智能很强,具体手法只能以"可选提示"
  出现,不得写成必须照做。方法的正确性建立在**结构**上(对抗验证环、agent 智能循环),
  不建立在按部就班使用某个手法上。
- **正交可复用**:每篇 method 收敛为一个正交能力,可脱离宿主任务独立使用
  (例:"循环对抗验收的抠图"与"对抗验收的拆帧"是两个正交方法,不与"制作角色动画"绑死)。
- **组合不强耦合**:高级 method = 引用/组合其它 method 与 recipe(走"向下指针"),
  不内联展开下层内容;强耦合(离开特定宿主任务即不成立)的方法不得直接入库,先解耦再收。

**写作高度纪律**(治理 run 裁剪判据):
- **删除测试**:这句话删掉,一个强 agent 会走弯路或踩死路吗?不会→删。
- **越权测试**:这句话在替 agent 做它自己能做对的决定吗?是→删。

## 2. 五个域(域轴)

`runtime` / `editor-tools` / `content` / `asset-pipeline` / `meta`(跨域:协作工作法、
交互面选择、通用验证法等)。每篇恰属一域。

目录即两轴直译:

```
agent_docs/<域>/norms.md
agent_docs/<域>/methods/<id>.md
agent_docs/<域>/mechanisms/<id>.md
agent_docs/<域>/recipes/<id>.md
agent_docs/<域>/decisions/<YYYY-MM-DD>-<id>.md
```

## 3. 收录标准(入库六条)

1. 只收跨实现演进仍成立的:不变量、指针、形状、配方、决策。
2. **每篇必须答得出"什么 agent、碰什么/做什么任务、为何必须读我"**——frontmatter 的
   triggers 填不出来的内容不配入库(decision 可豁免)。
3. 一事一篇:更新改原篇,禁止同主题新开(decision 除外,append-only)。
4. 机制卡限一页;method 骨架限一页(判断点/死路可略超)。
5. **不收**:时效性工作产物(计划/审查/任务书→`artifact/`)、可执行工具流程
   (→`.cursor/skills/`,其内嵌知识段应迁出成卡并被 skill 引用)、个人偏好(→agent 私有记忆)。
6. 索引一律生成(`INDEX.md`、`paths-triggers.json`),禁止手写;frontmatter 是唯一元数据源,
   字段规范见 [schema.md](schema.md)。

## 4. 两条治理管线

ground truth 位置不同,治理方式不同:

- **代码锚定**(norm 的事实部分 / mechanism / recipe):批量**盲重建对账**——治理 agent
  只读代码、禁止先读库内旧文档,从头推导机制图谱后逐条比对:属实/漂移/过时/缺失。
  盲推导是防确认偏误的关键,不可省。
- **实践锚定**(method / decision / norm 的规则部分):**事件溯源蒸馏**——证据源是
  `_meta/inbox/` 偏差记录、`artifact/` 实操产物、git 历史、会话转录。分诊三态:
  高频使用+零偏差=**静默确认**(盖章续期);偏差积压=**待蒸馏**(出改稿提案);
  久不使用=**标疑**(status: suspect,出"还留吗"审批题,不自动删)。

**偏差记录义务**(须写进各域 norm):任何 agent 在工作中发现现实与所用 method/卡片
冲突或超出时,收尾时向 `_meta/inbox/` 写一条偏差记录(格式见 inbox/README.md)。
写入零门槛、不需审批——这是实践锚定知识唯一的保鲜来源,错过现场即不可还原。

## 5. 审批面(压到最小)

**自动落地,不问人**:指针/锚点刷新;生成区块重生成;新增已知坑/死路(带证据引用,可逆);
补缺失机制卡;标记过时(status 变更);措辞裁剪(删除测试/越权测试);last_governed 盖章。

**必须上审批题**(仅四类):①不变量的增/删/改(给未来 agent 立法);②method 骨架变更
(增删阶段、改分工契约);③整篇文档删除或废弃;④与既有 decision 冲突的发现。

**审批题格式**:一行题干+一行证据+一行后果,yes/no 或三选一;无关键项则零提问静默完成。
禁止把报告甩给人读——**报告是给 agent 读的,人只收决策题**。

## 6. 与其它存放面的边界

- **CLAUDE.md / AGENTS.md**:目标形态是路由器(指向本库 INDEX)。已批准的接线(2026-07-11)
  仅限带标记的**开工闸门块**(`agent-docs-gate:begin/end`,由 cli.py install 维护);
  标记块之外的内容治理 run 仍不得擅改,全面路由器化(压缩正文)另须审批。
- **`.cursor/skills/`**:留"怎么做"(有步骤有工具调用);内嵌的"是什么/为什么"迁成机制卡后引用。
- **agent 私有记忆**:项目契约类内容升格入库、记忆留指针(已批准的迁移政策);个人偏好留记忆。
- **`artifact/`**:历史归档,治理 run 只从中蒸馏、不治理它。
- **`docs/`**:人看的设计文档;与本库重叠的部分迁入后旧址立牌(一行指针),不留双源。

## 7. 治理工具入口

- **治理台 CLI(统一入口)**:`python3 agent_docs/_meta/cli.py`——`list` 列全部治理流程、
  `route "业务一句话"` 现场匹配流程、`get <id>` 打印权威正文、`audit` 代理机械体检/索引/
  按改动路径查必读卡、`install` 一键安装/修复客户端薄壳(幂等,薄壳内容以 CLI 内模板为
  唯一生成源)。agent 遇到治理类业务先访问 CLI 现场发现流程,不凭记忆发挥。CLI 是无状态
  纯文件读取,**不开 server**——每次调用现读现返回,天然新鲜,不引入端口/生命周期/漂移。
- **流程注册即文件**:每个治理流程的唯一权威源 = `_meta/<id>-skill.md`(带 id/title/summary/
  triggers frontmatter,CLI 自动发现);现有:[governance-skill.md](governance-skill.md)(深度治理)、
  [intake-skill.md](intake-skill.md)(零散知识收编)。新增治理流程 = 新增该文件,零接线。
- **薄壳规则**:各客户端 skill 目录只装**一个** CLI 薄壳(`agent-docs-cli`),不再按流程逐个
  装壳;任何壳与 CLI 均不得复制流程正文。
- 每轮治理 run 结束必须追加 [_meta/governance-log.md](governance-log.md)(append-only;
  intake 收编记录也按行追加于此)。
