## 背景

故事仓(`../FindingDogStory/`)的故事设计目录新旧稿混放,多份已被 demo 制作侧取代;曾有 agent 误把新设计回写进 3 份死档,被 git 回退。2026-06-27 用户明确权威源并对账重写全部 Demo 制作资料(与实现 `flow_xungou_main` 对齐)。

## 决定

做寻狗内容/回写设计前先认四个权威源(均在 `../FindingDogStory/`):

- **主线(plot/flow)唯一权威 = `Demo制作资料/Demo完整流程_2026.06.05.md`**(锚文档,顶部是修订沿革 changelog + "主线/支线总览"表)。主线 11 拍 + 可选支线 3 件(枯井/义庄镇尸/城隍庙守夜,不挡通关不入主线编号);克拉拉招募解耦于义庄(结识在码头、招募是独立拍)。
- 阿秀 lore 权威 = `故事设计/阿秀支线-完整剧情与主线实施方案.md`(见 [冷框架决策](2026-06-27-axiu-signal-cold-framing.md))。
- 李天狗人物正传 = `故事设计/李天狗的故事.md`。
- demo 制作侧(关卡脚本/状态机/拆单/资产清单)= `Demo制作资料/` 整批。

**关键不变量:显示编号 ≠ 实现 s-id**。枯井/义庄降支线后主线显示号前移(如送货=显示⑧但 `s09_songhuo`、招募=显示⑨但 `s11_zhaomu`、终幕=显示⑩但 `s12_chufa`)——号→s-id 映射以锚文档总览表为准,别按显示号猜实现 id。

## 被否方案 / 死档(防翻案)

- **往旧稿回写设计**——否,曾因此回退。主线对不上时以 Demo完整流程 为准,不以故事设计旧稿为准。
- **死档(勿据此创作)**:`关二狗的故事(玩家视角)`、`设计分析/主线第九拍_非线性查访关卡设计`、`设计分析/查名关卡_实装脚本(demo)`及阿秀旧档——现已物理移入 `../FindingDogStory/废弃归档/`;**凡在废弃归档下的一律视为死档**。其中带回的过时设定(温情守护、复仇、李天狗读遗物认人)是污染源。
