## 背景

香粉味最初散落在过场里当一句 flavor("香粉味飘起"),无承载 channel、玩家看过就忘,落地失败。用户 2026-06-27 拍板把"气味"提成系统机制,设计已落盘 `../FindingDogStory/系统设计/气味系统设计文档.md`(权威),运行时已实现(SmellSystem,细节归 runtime 域)。

## 决定

气味做成一套**常驻、可学习、值驱动**的感官机制(同三把火/离死之距一档):HUD 气味丝常驻指示 + 嗅 action 主动玩法,不上全屏、不喊注意;世界观锚=煞机理"气是场",关二狗背尸匠的鼻子是对"气"世界的粗感知探头,香粉味=阿秀念气 crude 到连他都闻得到的那一处——一套系统同时焊住"关二狗本行"+"阿秀读气主题"。

**核心铁律(防跑偏)**:**系统常驻可学习 ≠ 香粉味常见**。气味 channel 天天用(尸臭/阴腥/香火/血),但香粉味仍是其中一个**稀有的冷读数**——濒死盲目显形、守冷框架、阿秀专属(见 [冷框架决策](2026-06-27-axiu-signal-cold-framing.md))。系统让它**可读**,不是让它**常见**;冷味一泛滥就废。

## 被否方案(防翻案)

- 香粉味继续散落过场当 flavor——已证落地失败。
- 因为"做成机制"就把香粉味撒到处都是——违铁律。
- 全屏表现/主动喊玩家注意——违三把火纪律。
- 无玩法用途的气味条目——每种味必须有线索/恐怖前奏/规矩用途,否则只是带 HUD 的 flavor。
