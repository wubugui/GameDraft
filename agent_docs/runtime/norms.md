# 游戏运行时开发规范

> 适用范围:所有运行时(`src/`)代码改动。
> 本规范只收录**跨架构演进仍然成立的不变量、过程义务与验收门**。具体架构、清单、字段、
> 步骤顺序一律不在此复述——那些以 `docs/游戏架构设计文档.md` 与代码为权威源,抄进规范
> 只会随实现漂移变成误导。
> **收录检验标准**:一条规范若依赖具体类名、方法名、步骤序或字段清单才能成立,它就是
> 实现描述,不收;类名在本文中只作举例(标"例"),不构成规范本体。

## 一、不变量(性质约束,违反即 bug,与具体实现无关)

1. **单一所有者**:任何一份运行时状态只有一个写入者;模态状态(面板可见性、游戏状态、
   返回栈)必须由同一个控制器原子管理,不允许 UI 与系统各改一半。
2. **失败不得伪装成功**:任何操作失败后,依赖其成功的后续写入(标记、任务进度、成功提示)
   不得发生——要么让失败可观测,要么补偿回滚(例:购买失败退款)。关键进度物不得因普通
   容量或资源限制被静默丢弃。
3. **异步必须封口**:每个 Promise 最终必须 resolve 或 reject,禁止永久悬挂;声明了顺序的
   动作批,处理器必须返回覆盖真实完成时间的 Promise,禁止 fire-and-forget 后立即报完成。
4. **旧时间线不写新状态**:读档、切场景、换会话、销毁之后,旧任务的写入必须被丢弃或隔离
   (epoch、静默清理、取消皆可,手段不限,性质必须成立)。
5. **生命周期对称**:destroy 不留残留(监听、定时器、RAF、资源、在途任务);
   destroy 后再 init,行为与首次启动一致。
6. **拒绝路径也要恢复**:任何 guard 拒绝、取消、异常分支,必须把玩家的可操作性恢复到
   进入该流程之前;只写 happy path 的流程不允许合入。
7. **构建期严于运行时**:数据错误尽量在编辑器与校验器侧 fail-closed(保存拒绝、校验报
   error);运行时对内容错误容错跳过,但 dev 下必须响(可见错误面),prod 下留痕。
8. **统一通道**:游戏行为经统一动作执行器;条件经统一求值器与唯一上下文工厂,禁止任何
   模块手工拼装缩水版上下文;运行时布尔/数值状态以统一存储为唯一真相源。
9. **登记面同步**:一个能力涉及的全部登记面(运行时注册、编辑器清单、校验器、参数往返
   保真登记)必须在同一次改动内改齐;权威清单在代码,以对账测试与校验器为准。
10. **资源有主**:每份资源(贴图、音频、缓存、监听)有明确所有者,所有者销毁时释放;
    清理完成后才结束的在途加载,其产物不得写回缓存。

## 二、过程义务(指路牌,不是地图)

1. **改前先读,只读相关**:改动某类机制前,先读架构文档中该机制的章节、该机制的权威源
   代码、以及本库/审查包中的对应条目;不要求通读无关部分。
2. **扩展走既定流程**:新增能力原语(新动作、新参数、新条件叶子等)走项目既定扩展流程
   (例:add-game-action),完成后明确报告改动了哪些登记面。
3. **拿不准就停**:无法在不违反不变量的前提下实现时,停下报告,不得绕过机制糊弄。
4. **行为改动真机验证**:行为类改动在测试之外须用真机手段验证到"看见行为正确"
   (命令通道/页内驱动等既有配方),不以"编译过 + 测试绿"为完成标准。
5. **偏差记录义务**:工作中发现现实与本库文档(规范/工作法/机制卡/配方)打架或超出时,
   收尾时向 `agent_docs/_meta/inbox/` 写一条三行偏差记录(格式见该目录 README;
   零门槛,不需审批)。这是库内实践类知识唯一的保鲜来源。

## 三、验收门(每次运行时合入,机械执行)

- `npx tsc --noEmit` 零错误;
- `npm test` 全绿(仅当前工作区,不含隐藏 worktree);
- `./dev.sh validate-data` 零 error,且警告数不增加;
- 素材引用审计零问题;
- `git diff --check` 干净。

## 四、审批红线(出现任一,拒绝合入)

- 玩家输入或 UI 状态可能永久锁死;
- 恢复后的状态可能被旧时间线覆盖;
- 关键道具、货币或进度可能失败却继续推进;
- 存在可能永久悬挂的 Promise;
- 新增系统缺完整销毁逻辑;
- 登记面(运行时/编辑器/校验器)未同步;
- 新增 validate-data error,或使警告数上升。

## 附:已确认的默认决策(2026-07-11)

- **未知动作**:构建期 fail-closed;运行时 dev 必须响、prod 跳过并留痕(不采用 prod fail-closed)。
- **关键给予**:关键进度物绕过普通容量限制(已以 giveItem critical 参数实现)。
- **警告棘轮**:合入不得新增严格校验警告;存量警告白名单/清偿机制另立项,落地前不启用
  strict 模式作硬门禁。
