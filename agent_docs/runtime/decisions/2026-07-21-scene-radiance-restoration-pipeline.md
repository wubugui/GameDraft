## 背景

二维绘制场景只有 LDR 颜色，实体素材又必须一概作为 albedo 进入 Lambert 管线；若不先恢复场景辐射度，场景图上画出的灯、窗与明暗关系无法成为实体的入射光。纯亮度规则会把白石、道路和水面误判成发光体，纯语义模型又能认出“灯笼”却不能可靠判断它是否点亮，因此二者都不能单独生成发光增益图。

## 决定

场景辐射度还原固定为离线混合管线：LDR 先经 sRGB EOTF 变为线性基础辐射，每张图必须有独立 `sceneExposureEV`；开放词汇分割模型只产生潜在蜡烛、灯笼、窗与开口面的候选掩码，不直接决定增益；最终局部 `gainEV(x)` 必须同时通过像素局部对比、候选物内部亮度相对远环、八方向近环溢光和全图环境能量先验的检验。还原式为

`C_hdr(x) = 100 × EOTF_sRGB(C_ldr(x)) × 2^(sceneExposureEV + gainEV(x))`，

局部增益上限可配置，首轮参考值为 `+3.32 EV`。模型仅参与开发期离线烘焙，运行时只消费辐射度/增益产物；随后所有实体受光必须在深度还原的伪世界空间内，从该场景辐射场积分 irradiance，禁止让屏幕空间重叠但伪世界深度相离的像素互相照明。

## 职责边界

- `sceneExposureEV` 决定整张场景的基础环境亮度，允许逐图人工调节，但不负责标发光体。
- 语义分割只回答“哪里像潜在光源载体”，不能用模型分数替代“是否正在发光”。
- 辐射证据回答“候选物是否具有高于环境的能量与空间溢光结构”，并生成稀疏 `gainEV`。
- HDR 场景辐射度是伪世界 path tracing / irradiance cache 的入射辐射源，不等于已经积分后的实体 irradiance。
- 深度、法线和实体 quad 必须还原到同一伪世界坐标约定后再做遮挡与积分。

## 被否方案

- 由通用 agent 逐图目测标发光增益：不可复现，难以批量回归。
- 纯统计亮度或固定 LDR→HDR 肩线：会把日景白石、亮路与水面抬成发光体。
- 让语义模型直接输出最终增益：模型无法可靠区分点亮与未点亮灯具，也会把整个灯具/窗框当辐射核心。
- 逐场景堆专用手写规则：维护成本随场景数线性增长，违背统一模型目标。
- 在屏幕空间直接模糊或积分：无法排除屏幕重合、伪世界深度相离的错误传光。
- 在游戏运行时携带分割模型：模型体量与内存成本没有必要，离线烘焙即可。

## 验收口径

- 正样本必须保留灯、烛火、发光窗口等辐射核心；负样本必须覆盖未点亮灯具、白石、亮路、水面和雾中高亮。
- 不能只报告“场景内检出过光源”；必须同时审查漏检、误检、局部增益峰值、增益覆盖率和逐图曝光。
- 模型候选分数若在正负样本间重叠，禁止靠调单一阈值假装收敛，必须交给辐射证据裁决。
- 进入实体着色验证前，典型场景必须先输出带 nit/EV 标尺的 HDR 辐射度图、数值统计与中间增益图。
