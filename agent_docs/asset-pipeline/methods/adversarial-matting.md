## 适用时机

任何"把主体从背景里分离出来"的任务:角色动画帧、对话立绘、视差图层、烤入人物……
不限于角色动画。本卡管**工作形状与判断**,具体手法(用哪条抠图路线)见向下指针。

> 这是一个正交能力,可脱离宿主任务独立使用;高级任务(如 [角色动画生产](character-animation-production.md))
> 引用它,而不内联它。

## 阶段骨架(每阶段:目的 + 完成判据,不写怎么达成)

1. **定验收疆域**——目的:把"抠干净"翻译成可判的黑白。完成判据:四条验收定义写死——
   ①无背景残留(halo);②不多扣(不切进主体);③主体完整(该在的部件都在);
   ④主体真实的**可透空隙**(飞袍与腿间、持械三角等)保留为透明。
2. **出一版分离**——目的:得到一版 alpha + 去污后的边缘。完成判据:主体大体覆盖到位。
   手法可选(见向下指针),不在本卡规定。
3. **对抗验收**——目的:主动证伪这一版(去找它最可能翻车的帧/部位),而非自证通过。
   完成判据:在**正确的层面**度量后仍找不到反例(见判断点)。
4. **诊断再出**——目的:验收不过时,先定位是四类失败中的哪一类,再针对性修,回到 2。
   完成判据:循环到验收通过为止。

## 判断点(哪里要现场判断,拿什么证据判)

- **真背景空隙 vs 过扣洞(最关键)**:二者都表现为"被主体包围的透明区",
  **颜色分不了**(同色衣≈同色底)、**拓扑也分不了**(两者都封闭)——只有**物体先验**
  (神经分割的置信度)分得清:先验判该处是物体 = 过扣、要回填;判是背景 = 真空隙、要留。
  判据永远落在物体先验上,不落在颜色/形状上。
- **度量要落在对的层面**:多扣要对着物体先验在**源级**测(先验高置信却被抠掉的比例),
  **不能**用成品上"封闭透明=洞"当指标——那会把上面的真空隙全部误报成多扣。
- **无 halo 的判读**:放到纯色底/多底上肉眼查边缘残留(灰边);量化 halo 指标单独**不可裁决**
  (会误报浅色发丝等),须人/agent 目视复核。

## 分工契约(程序做什么 / agent 裁什么 / 人拍什么板)

- **程序**:出 alpha、去污、算候选指标(残留/多扣/碎片)、合成多底质检图。
- **agent**:定验收疆域、对抗证伪(主动挑最险的帧/部位)、裁定真空隙 vs 多扣、决定是否再循环。
- **人**:拍板"这个主体的边界/取舍"这类审美或语义争议(如白发丝、半透明道具算不算)。

## 已知死路(链 decision / 坑)

- **靠腐蚀边缘去 halo = 多扣**(错路);去 halo 的正解是**去污**(改半透明边像素的颜色、不动 alpha)。
- **用成品"封闭透明=洞"当多扣指标** → 真空隙全误报,一批干净结果被误判失败。→ 必须源级测。
- 判读细节与仓库现有抠图路线见 [抠图路线与判读铁律](../mechanisms/matting-toolbox.md)。
