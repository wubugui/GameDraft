## 背景(一段)

2026-07-04 做 dev_teahouse_alive"让烤进原画的坐姿茶客动起来",连续三轮被用户当场打脸
("缺部分/位置漂/被裁")后收敛出的技术路线。难点:画师把人和桌椅"融"在一起画,
抠图与底图方案互相牵制。工作法骨架见
[烤入背景人物活化](../methods/baked-figure-activation.md)。

## 决定(一句)

底图 = **真原画 + 只擦被活化者剪影**(擦洞用空背景像素填充);抠人 = **BiRefNet 整坨 +
手画"包人排家具"保留多边形**(多边形画在人轮廓外侧,只切家具、永不切人);动作 =
**只向上鼓的升余弦呼吸**(`1+heave*(1-cos)/2`,永不小于原尺寸,sway=0)。

## 被否方案

- **整图重画的空背景当底**(image2image 移除所有人):重画的桌子整体偏移几像素,
  抠图按原图坐标摆→截断边对不上=角色像被裁一截;且人全删后抠图没含到的部位(手/碗/下半身)直接消失。
- **形态学开运算(MORPH_OPEN 大核)+ 取最靠上大块**分离人与家具:会把人自己的脖子/腰砍断,
  只剩"头+碗"漂浮碎片。
- **对称 sin 呼吸 / 横向 sway**:缩放的收缩半程会让 overlay 比擦洞小、露出洞里填的背景;
  sway 同理滑出洞边。
- **nebula(LibTV)inverse isolate 单独抠人**:人漂空 + 描边 halo + 内容被重绘。
